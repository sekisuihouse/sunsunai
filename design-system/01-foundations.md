# 01. Foundations

## 観測された事実

### 色

本体サイトは白背景、黒文字、黒枠を中心にしたモノクロ基調です。`body` は `background:#ffffff`、本文色は `#000000`、主要ボタンやリンクカードは `#000000` の 4px 枠を使います。根拠: EV-0001, EV-0003, EV-0012, EV-0017

薄い区切りや背景には `#e5e5e5`、`#ececec`、`#f7f7f7` などのグレーが繰り返されます。根拠: EV-0019, EV-0031

LP/キャンペーンは本体と別系統です。学校説明会 LP は `#58C3F5` と `#92DF55`、未来の学校 FES は `#243FAB`、`#00A0E9`、`#E4007F`、`#FFE100` を使います。根拠: EV-0026, EV-0027, EV-0031

### タイポグラフィ

本体の日本語フォントは OS 標準の sans-serif stack です。`font-weight:500` が body 周辺の基本、見出しやナビ/ボタンは bold が多用されます。根拠: EV-0001, EV-0004, EV-0010, EV-0012

本文の基準は `20px`、`line-height:2`、`letter-spacing:.05em` のゆったりした設定です。パンくずや補助ナビは `11px` から `12px`、ナビは `15px` から `16px`、リンクカード見出しは `30px` です。根拠: EV-0004, EV-0010, EV-0017, EV-0019

`futura-pt-bold` は CSS 上で参照されますが、ミラー内にローカルフォントファイルはありません。配信元やライセンスは未確認です。根拠: EV-0030

### レイアウト

本体のコンテナは `1024px`、内部コンテナは `812px` を基準にします。`1023px` 以下では `calc(100% - 50px)`、`374px` 以下では `calc(100% - 30px)` のように左右余白が縮みます。根拠: EV-0005, EV-0032

ヘッダーは `1204px` 以上で左サイドレール型になり、幅 `180px`、高さ `100vh` です。それ未満では上部 fixed header とメニューボタンを使います。根拠: EV-0007, EV-0008, EV-0035

### 余白・サイズ

主要な構造値は以下です。

| Token | Value | 用途 | 根拠 |
| --- | ---: | --- | --- |
| `size.container` | `1024px` | 本体コンテナ | EV-0005 |
| `size.container-inner` | `812px` | 内部コンテンツ | EV-0005 |
| `size.desktop-rail` | `180px` | デスクトップ左ナビ | EV-0007 |
| `size.button` | `312px` | 主要ボタン幅 | EV-0012 |
| `size.button-mobile` | `184px` | SP 主要ボタン幅 | EV-0015 |
| `space.5` | `20px` | ボタン padding | EV-0012 |
| `space.15` | `60px` | 見出し下/フッター SP | EV-0004, EV-0011 |
| `space.25` | `100px` | ヘッダー高/フッター上余白 | EV-0007, EV-0011 |

### 角丸・枠線・影

本体の主要 CTA は角丸よりも黒い太枠と反転アニメーションが強い識別要素です。ボタン/リンクカードの枠線は `4px`、SP ボタンは `2px` です。根拠: EV-0012, EV-0015, EV-0017

頻出角丸は `4px`、`6px`、`10px` です。円形マスクや背景アニメーションには `50%`、`999rem` が使われます。根拠: EV-0009, EV-0014, EV-0016, EV-0039

影は `0 4px 15px #0000001a` が最も安定した候補です。根拠: EV-0039

### Motion

基本の transition は `all .35s ease-in-out` です。メニューボタン、ボタン、hover mask などに広く使われます。根拠: EV-0008, EV-0039

ページ遷移は `.6s`、reveal は `.8s ease-in-out` で opacity と translateY を制御します。根拠: EV-0020, EV-0037

### Breakpoints

本体サイトの主な breakpoint は以下です。

| Token | Value | 用途 | 根拠 |
| --- | ---: | --- | --- |
| `mobile-narrow` | `374px` | 狭い SP の余白調整 | EV-0032 |
| `mobile` | `767px` | `.sp_only`/`.pc_only` 切り替え | EV-0006 |
| `tablet-min` | `768px` | desktop hover mask 有効化 | EV-0016 |
| `tablet` | `1023px` | コンテナ幅調整 | EV-0032 |
| `desktop-rail` | `1204px` | 左サイドナビ化 | EV-0007 |

LP では `750px`、`1080px` も使われるため、本体 breakpoint と分離して扱う必要があります。根拠: EV-0032

## 複数証拠から推論した規則

1. 本体サイトの核は「白い余白、黒線、太字、日本語本文のゆったりした行間」です。根拠: EV-0001, EV-0003, EV-0004, EV-0012, EV-0017
2. コンポーネントの識別は色数ではなく、黒枠、円弧/円形マスク、太い文字、広い余白で作られています。根拠: EV-0009, EV-0014, EV-0016, EV-0035
3. LP は本体デザインシステムの拡張テーマではなく、キャンペーン固有 theme として分離する方が安全です。根拠: EV-0026, EV-0027, EV-0031

## 改善提案

1. 本体 token と campaign token の namespace を分ける。
2. `767px` と `750px` のような近接 breakpoint を役割別に整理する。
3. `outline:none` を廃止または代替 focus ring token に置き換える。根拠: EV-0002
4. `futura-pt-bold` の配信元とライセンスを確認し、使用するなら font token に正式登録する。根拠: EV-0030
