プロジェクト名
授業改善支援システム（リアルタイムコメント・AIフィードバック）

1. 概要
対面授業において、学生同士の会話や感想をリアルタイムで収集・文字起こしし、授業画面にニコニコ動画風のコメント（弾幕）として表示する。

授業終了後には、学生コメント・教員発話・画面録画をもとにAIが授業改善レポートを生成し、教員が授業を振り返りやすくする。

2. 解決したい課題
学生の率直な意見を教員が把握できない

授業中の反応を後から分析できない

授業改善が教員の記憶頼りになっている

学生の理解度や興味の変化が分からない

11. 開発方針（Codex向け）
MVPを優先し、まずは「授業開始 → 音声取得 → リアルタイム文字起こし → 弾幕表示 → タイムライン保存 → AIレポート生成」までを実装する。

コンポーネント指向で実装し、音声取得・文字起こし・リアルタイム表示・AI分析を疎結合に設計する。

AI分析機能は将来的に分析項目を追加しやすいようモジュール化する。

UIは教員が授業中に一目で状況を把握できることを重視し、ライブ表示と授業後の振り返り画面を分離する。

12. UIデザインシステム指令
このアプリのUIは、リポジトリ内の /design-system を唯一のデザイン仕様として使用して作成する。

実装前に必ず以下を読む。

AGENTS.md

/design-system/DESIGN_SYSTEM.md

/design-system/tokens.json

/design-system/components.json

/design-system/QUALITY_REPORT.md

/design-system/evidence.json

UIを作るときは、見た目を雰囲気で真似るのではなく、tokens.json と components.json を実装へ構造的に反映する。

色、文字サイズ、行間、余白、枠線、角丸、影、アニメーション時間、ブレークポイントは、デザインシステムのトークンまたはCSS変数から使う。

任意の色、任意の余白、任意の角丸、任意のフォントサイズをハードコードしない。

本体画面は、白背景、黒文字、黒枠、太字の日本語、広い余白を中心としたコアサイトのデザインを使用する。

キャンペーンLP用の色や表現は、本体画面に混ぜない。必要な場合のみ campaign theme として分離して使う。

主要UIは、components.json に定義された以下のコンポーネント方針を優先して再利用・実装する。

GlobalShell

HeaderNavigation

OverlayNavigation

Button

HoverMaskLink

アプリ内には central token layer を作成し、/design-system/tokens.json を CSS変数、テーマ設定、または同等の仕組みに変換して使用する。

アクセシビリティとして、semantic HTML、キーボード操作、visible focus state、prefers-reduced-motion を考慮する。

UI実装後は、使用したトークン、再利用または追加したコンポーネント、仕様との差異、情報不足による仮定を報告する。

13. 弾幕表示と拡張機能の許可
授業中のコメント表示は、アプリ内のライブビューに弾幕として表示するだけでなく、必要に応じてブラウザ拡張機能を使ってWeb画面上に文字を重ねて表示してもよい。

拡張機能を使う場合は、Chrome Extension などの content script により、Google Meet、授業資料、ブラウザ上の共有画面、または指定したWebページの上に、ニコニコ動画風の弾幕コメントをオーバーレイ表示できる設計にする。

弾幕オーバーレイは、授業資料の可読性を妨げないように、上部・下部・透明度・文字サイズ・表示速度・一時停止・非表示を調整できるようにする。

拡張機能は必須ではないが、Web画面全体に弾幕を重ねたい場合の実装手段として許可する。

拡張機能を実装する場合も、デザインシステムのトークンを使い、アプリ本体と視覚ルールを揃える。
